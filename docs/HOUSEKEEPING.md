# Project Housekeeping

This file defines what belongs in Git and what should stay local.

## Git-Tracked Files

- Application source code under `src/`, `Flask/`, and other maintained code directories.
- GitHub-safe documentation under `docs/`.
- Reproducible curation outputs: notes, reports, manifests, extraction TSV files, SQL drafts, and scripts.
- Small public assets required by the frontend.

## Local-Only Files

These are intentionally ignored and should not be committed:

- `.env`, `Flask/.env`, and other local secrets.
- `node_modules/`.
- `data/tmp/`: staging area for manually downloaded papers and supplementary files.
- `field-curation-workdir/`: local full curation workspace with raw literature, database snapshots, OA packages, and large intermediate files.
- `dist/`: frontend build output generated by `npm run build`.
- Python caches, pytest caches, backend logs, and Flask export cache files.

## tRNAtherapeutics Manual Downloads

Manual literature downloads should be staged as:

- `data/tmp/{PMID}/fulltext.pdf`
- `data/tmp/{PMID}/supplement_1.pdf`
- `data/tmp/{PMID}/supplement_2.xlsx`
- `data/tmp/{PMID}/supplement_3.docx`

After format validation and curation review, verified raw files can be copied into:

- `field-curation-workdir/full_tRNAtherapeutics/papers/{PMID}/`
- `field-curation-workdir/full_tRNAtherapeutics/supplementary/{PMID}/`

Only derived, GitHub-safe artifacts should go under `docs/curation/tRNAtherapeutics/`.

## Legacy Local Workspace

`python scripts/` is an ignored legacy workspace. It contains old notebooks, CSV outputs, images, AF3-related scratch files, and generated JSON. Do not delete it as generic trash without a separate inventory pass. New reusable scripts should go under `tools/`, and new curation documentation should go under `docs/curation/`.

## Cleanup Checklist

Before committing:

1. Run `git status --short --untracked-files=all`.
2. Remove obvious generated junk: caches, logs, temporary export files, and build output.
3. Keep raw papers and local database snapshots out of Git.
4. Run `git diff --check`.
5. Run the relevant build or test command for touched code.
6. Scan staged curation files for secrets before committing.
