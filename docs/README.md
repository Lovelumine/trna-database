# Documentation Layout

This repository keeps documentation in three buckets:

- `README.md`: repository entry and setup notes.
- `docs/HOUSEKEEPING.md`: local file policy, ignored workspace rules, and cleanup checklist.
- `docs/plans/active/`: active internal implementation plans that still guide ongoing work.
- `docs/curation/`: GitHub-safe curation notes, reports, manifests, SQL drafts, and scripts.
- `public/docs/`: user-facing help content shipped with the site.

Completed, redundant, or stale internal plan files should be removed instead of accumulating in the repository root.
